# superpowers-lite skills

Skills are installed at `<SKILLS_PATH>`. Each is a folder with a `SKILL.md`.

When the user asks about any of these topics, start by reading the
matching SKILL.md before responding:

- Debugging, test failures, unexpected behavior → `<SKILLS_PATH>/systematic-debugging/SKILL.md`
- Brainstorming new features / requirements      → `<SKILLS_PATH>/brainstorming/SKILL.md`
- Turning a spec into a plan                      → `<SKILLS_PATH>/writing-plans/SKILL.md`
- Executing an existing plan                      → `<SKILLS_PATH>/executing-plans/SKILL.md`
- Multi-step implementation with subagents        → `<SKILLS_PATH>/subagent-driven-development/SKILL.md`
- 2+ independent tasks that can run in parallel   → `<SKILLS_PATH>/dispatching-parallel-agents/SKILL.md`

The entry-point skill `<SKILLS_PATH>/using-superpowers/SKILL.md` explains how
to find and use the others — read it first whenever any of the conditions
above apply.
